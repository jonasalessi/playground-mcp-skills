# Overview
This project is about to compare the efficiency of using mcp or skill when reading a jdbc log metrics.

### Code Style & Formatting

**Functions & Architecture**
- Keep functions small, focused, and concise.
- Minimize nesting by using early returns.
- Separate concerns clearly.
- Prefer composition over inheritance.
- MUST NOT create code that will not be used.

**Naming Conventions**
- Keep variable and function names descriptive but concise (max 30 characters).
- Write meaningful, self-documenting code.
- Function and method names MUST start with a verb that clearly describes the action.
- Function and method names MUST NEVER start with a noun.

**Comments**
- MUST NOT add comments to explain HOW the code works, only explain WHY.

### Commands
- Build: `npm run build`
- Format: `npm run format`
- Lint: `npm run lint`
